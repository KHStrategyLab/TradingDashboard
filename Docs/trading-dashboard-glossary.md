# TradingDashboard Glossary

이 문서는 TradingDashboard에서 반복해서 쓰는 용어를 고정하기 위한 단어사전이다.

같은 단어를 다른 뜻으로 쓰면 KRX/NXT 기준, 기준봉, 전략 판단, 화면 표시가 섞일 수 있으므로 새 기능을 추가할 때 이 문서를 먼저 확인한다.

## 기준봉 계열

### 기준봉

KRX 정규장 일봉 기준봉을 뜻한다.

- NXT 종목이어도 기준봉은 KRX 일봉 기준이다.
- 기준봉 판단은 KRX 정규장 일봉을 우선한다.
- SOR/NXT 흐름은 버리지 않고, 당일 흐름 확인과 보조 판단에 따로 사용한다.
- `BaseCandle*`, `BaseHalfPrice`, `BasePrice` 계열 필드는 이 기준봉을 의미한다.
- 장중 1분봉 장대양봉을 기준봉이라고 부르지 않는다.

### 기준봉 게이트

조건식 편입 종목을 화면 목록에 올리기 전에 거치는 1차 문이다.

- 검사 범위: 0봉전 기준 최근 10거래일, 즉 0봉전~9봉전
- 기본 조건: 거래대금 1,000억 이상 AND 전일대비등락률 20% 이상
- 통과하면 왼쪽 목록에 표시하고 `GateBaseCandle*` 필드에 결과를 저장한다.
- 초기 조건식 결과에서 실패하면 화면에 표시하지 않는다.
- 장중 실시간 신규 편입에서 실패하면 `NEW`로 화면에 표시한다.
- 실패 결과는 캐시에 저장하지 않는다. 같은 종목이 이후 조건식에 다시 편입되면 다시 검사한다.
- 이 게이트는 매수 신호가 아니라 "볼 종목인지"를 거르는 입장권이다.

### 게이트 기준봉

`기준봉 게이트`를 통과시킨 일봉이다.

- 기존 KRX 전략 기준봉 `BaseCandle*`, `BaseHalfPrice`와 구분한다.
- 저장 필드 예: `GateBaseCandleOffset`, `GateBaseCandleDate`, `GateBaseCandleMarket`, `GateBaseCandleChangeRate`, `GateBaseCandleTradeValue`
- `GateBaseCandleOffset`은 파일 캐시에 저장한다. 재시작 후에도 `D+` 배지를 복원하기 위한 값이며, 기존 캐시에 값이 없으면 기본값 `-1`을 유지한다.
- 목록 배지 예: `D+0`, `D+3`
- `D+3`은 기준봉 발생 후 3거래일째 추적 중이라는 뜻이다.
- 게이트 시장은 `GateBaseCandleMarket`에 저장만 하고 화면에는 표시하지 않는다.
- 시장구분과 NXT 가능 여부는 종목 자체의 기존 시장/`NX` 표시가 담당한다.
- 기준봉 화면 표시는 날짜 진행 의미와 기준봉 수치만 맡는다.
- 전략 진행률에서는 초기 문 또는 기준 확인 단계로 사용할 수 있다.

### 장중 신규 편입

장중 조건검색 실시간 편입으로 새로 들어온 종목 중, 기준봉 게이트를 통과하지 못했지만 화면에서 관찰할 종목이다.

- 초기 조건식 결과의 게이트 실패 종목은 장중 신규 편입이 아니다.
- 장중 실시간 편입 종목만 이 배지를 받을 수 있다.
- 화면에는 파란 `NEW` 배지를 붙인다.
- 파일에는 저장하지 않는다.
- 0B 등록 대상에는 포함한다.
- 매수 가능 여부는 목록 레이어가 판단하지 않고 전략실이 판단한다.

### D+ 배지

기준봉 게이트를 통과한 종목에 붙는 빨간 배지다.

- `D+0`: 당일 기준봉 게이트 통과
- `D+1`: 기준봉 다음 거래일
- `D+2`: 기준봉 이후 2거래일째
- 추적 만료 기간은 프로그램 소스가 아니라 조건검색식이 정한다.
- 프로그램은 조건검색식이 내려준 오늘 후보 안에서 `D+` 배지를 표시한다.
- 당일 기준봉 게이트 통과 `D+0`은 더 강한 빨강으로 표시한다.
- 장중 신규 편입과 기준봉 게이트가 동시에 성립하면 `NEW`와 `D+0`이 함께 표시될 수 있다.

### 기준봉 게이트 거래대금

기준봉 게이트의 거래대금은 원 단위로 비교한다.

```text
거래대금 = 종가 * 거래량
기준값 = 100,000,000,000원 = 1,000억
```

`ka10081` 일봉 응답의 `trde_prica` 계열 거래대금 필드는 TR/응답에 따라 원 단위가 아니라 백만원 단위 또는 축약 단위처럼 들어올 수 있다.

따라서 기준봉 게이트에서는 `trde_prica`를 그대로 1,000억 기준과 비교하지 않는다.

현재 구현은 다음 원칙을 따른다.

```text
API 거래대금 후보 = trde_prica 계열 필드
추정 거래대금 = 종가 * 거래량
게이트 비교값 = Max(API 거래대금 후보, 추정 거래대금)
```

이렇게 해야 삼성에스디에스처럼 실제 거래대금이 1,000억을 넘는 종목이 단위 문제로 버려지지 않는다.

### 일봉 기준봉

`기준봉`과 같은 뜻이다.

혼동이 생길 때는 `KRX 일봉 기준봉`이라고 쓴다.

### 장중 기준봉

당일 장중 분봉에서 생긴 강한 봉을 뜻한다.

- 예: 당일 1분봉 +2% 이상 장대양봉
- 기존 `BaseCandle*` 필드를 덮어쓰지 않는다.
- 장중 눌림/반응/확인용 임시 기준으로만 사용한다.

### 상한가 기준봉

전일 또는 기준일 KRX 일봉에서 상한가급으로 강하게 오른 기준봉이다.

- 전일 종가등락률 28% 이상 후보를 우선 본다.
- 몸통 눌림, 시가 회복, 탈출봉 고가 돌파 전략과 연결된다.

### 기준가

화면과 전략에서 색상/등락률/기준 비교에 쓰는 가격이다.

- 원칙: KRX 전일종가
- NXT 현재가, NXT 종가, `_AL`, 0B 실시간값이 덮어쓸 수 없다.

### BaseHalfPrice

기준봉 보조 기준값이다.

```text
BaseHalfPrice = (전일종가 + 기준봉 종가) / 2
```

기존 KHStrategyLab 기준을 보존한다.

### BaseBodyHalfPrice

기준봉 몸통 중심값이다.

```text
BaseBodyHalfPrice = (기준봉 시가 + 기준봉 종가) / 2
```

상한가 몸통 눌림 판단에서 참고할 수 있다.

## 차트 계열

### 화면 일봉

사용자가 화면에서 보고 있는 일봉 차트다.

- KRX 시간대에는 KRX 일봉일 수 있다.
- NXT 시간대에는 NXT 일봉일 수 있다.
- 표시용 차트이며, 전략 기준봉 판단용으로 그대로 쓰면 안 된다.

### 판단 일봉

전략 판단에 쓰는 일봉이다.

- 기준봉 판단용 판단 일봉은 KRX 정규장 일봉을 우선한다.
- SOR/NXT 일봉은 당일 흐름과 보조 확인용으로 분리해서 본다.
- 화면 일봉과 다를 수 있다.

### 분봉

1분, 3분, 5분, 10분, 15분, 30분, 60분, 120분 차트를 뜻한다.

분봉은 시장 기준을 반드시 함께 표시한다.

```text
KRX 10분봉
NXT 10분봉
_AL 통합 10분봉
```

### 0봉

현재 진행 중인 봉이다.

- 실시간 값이 반영될 수 있다.
- 확정봉이 아니므로 전략 조건에서 사용할 때 주의한다.

### 1봉전

가장 최근에 완료된 봉이다.

- 확정된 봉으로 본다.
- 전략 판단에서 기본 기준으로 쓰기 좋다.

## 시장 계열

### KRX

한국거래소 정규장 기준 데이터 또는 주문 시장이다.

### NXT

넥스트트레이드 기준 데이터 또는 주문 시장이다.

- NXT 시가/고가/저가/종가와 현재가는 화면 표시에서 NXT 값을 쓸 수 있다.
- 기준가와 기준봉은 NXT 값으로 덮지 않는다.

### SOR

주문 라우팅 방식이다.

- 주문 요청에서는 `dmst_stex_tp = SOR`를 사용한다.
- 주문 종목코드는 6자리 종목코드를 사용한다.
- `_NX`, `_AL` suffix를 주문 종목코드에 넣지 않는다.

### _NX

NXT 시장 데이터 조회용 종목코드 suffix다.

예:

```text
018260_NX
```

주문 종목코드에는 사용하지 않는다.

### _AL

통합 또는 MTS SOR ON 표시와 맞추기 위해 쓰는 시장 데이터 조회용 suffix다.

- 거래량, 거래대금, 프로그램, 일별거래상세 보정에 사용할 수 있다.
- 주문 종목코드에는 사용하지 않는다.
- 분봉/차트 판단에 쓸 수 있는지는 실제 응답 검증이 필요하다.

## 전략 계열

### 조건00 MA 회복초입

기존 KHStrategyLab 주전략이다.
현재 TradingDashboard의 `Kiwoom.ConditionSeq01` 설정값을 뜻하는 말이 아니라, 기존 전략 문서에서 쓰던 전략명이다.

```text
KRX 일봉 기준봉
→ 기준봉 이후 10분 MA60 이탈
→ 10분 MA60 회복
→ 10분 MA5 > MA20 / MA60
→ 5분 20봉 신고가
```

### SOR 당일강세 눌림 돌파

당일 20% 이상 강세와 거래대금 1,000억 이상을 전제로 보는 SOR 계열 전략이다.

안정형:

```text
15분봉 60선 눌림 + 5분봉 20봉 신고가
```

중간형:

```text
10분봉 60선 눌림 + 5분봉 20봉 신고가
```

공격형:

```text
10분봉 60선 눌림 + 3분봉 20봉 신고가
```

### 후보패턴

전략 슬롯에 올리기 전의 관찰용 패턴이다.

- 알림 검증 또는 수동 관찰용
- 주문 연결 없음
- 예: 당일 1분 장대양봉 눌림

## 진행률 계열

### 진행률

전략의 전체 사이클 진행 상태다.

100%는 매수 시점이 아니라 매도 완료 후 전략 종료를 뜻한다.

### 매수 구간

0~70% 구간이다.

검색식 편입, 기준 확인, 눌림, 회복, 돌파, 매수 후보까지 포함한다.
전략별 단계 수가 달라도 이 구간 안에서 자동 분배한다.

### 매도 추적 구간

70~100% 구간이다.

보유 진입, 1차 목표, 손절, 트레일링, 매도 완료까지 포함한다.

## 뉴스/공시 계열

### 뉴스 필터

뉴스 제목/요약에서 단어사전을 이용해 필요 없는 뉴스를 걸러내는 보조 장치다.

매수 신호가 아니다.

### 공시 필터

조건식 편입 후보에 대해 좋은 재료 공시를 우선 확인하는 보조 장치다.

전체 위험 공시 감시기가 아니다.
