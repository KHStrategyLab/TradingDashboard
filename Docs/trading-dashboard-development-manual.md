# TradingDashboard 개발 매뉴얼

작성일: 2026-05-29

이 문서는 TradingDashboard를 계속 개발할 때 흔들리지 않기 위한 기준 문서다. 화면의 각 영역이 어떤 REST TR, WebSocket 실시간 항목, 캐시, 계산식으로 채워지는지 기록한다. 기능을 추가하거나 수정할 때는 이 문서를 먼저 보고, 새 화면 항목이 생기면 데이터 출처와 갱신 규칙도 같이 업데이트한다.

## 1차 목표

이번 프로그램의 1차 목표는 호가창 완성이다.

- 키움/MTS에서 보는 호가, 체결, 거래량, 거래대금, 등락률, 기준가 관련 수치와 화면 값이 일치해야 한다.
- 매수/매도 호가, 잔량, 현재가, 체결량, 매수체결량, 매도체결량은 실시간과 REST 스냅샷을 조합해 MTS 기준으로 검증한다.
- KRX/NXT 시간대가 달라도 기준가와 색상 기준은 흔들리지 않아야 한다.
- 장중에는 WebSocket 실시간 값을 우선하고, 장마감 이후에는 REST 최종 스냅샷으로 빈 화면 없이 복원한다.
- 수치가 MTS와 다르면 UI 모양보다 데이터 출처, TR 필드, 계산식을 먼저 의심한다.
- 호가창과 오른쪽 체결/종목정보 수치가 안정된 뒤에 추적기, 검색식 확장, 매매 판단 보조 기능을 붙인다.

## 1차 달성 기록

2026-05-29 기준, 하루 작업으로 1차 목표의 핵심 골격을 완성했다.

- 조건검색식 01번에서 종목 목록을 받아 왼쪽 목록에 표시한다.
- 조건검색 실시간 추적으로 장중 편입/이탈을 처리한다.
- 선택 종목의 KRX 전일종가를 기준가로 잠그고, 화면 색상 기준으로 사용한다.
- 선택 종목의 호가창을 `0D/0H` 실시간과 REST 스냅샷으로 구성한다.
- 관심종목 전체의 `0B` 체결 실시간을 받아 현재가, 거래량, 체결 정보를 움직인다.
- 오른쪽 체결/종목정보 영역에 MTS와 맞춰야 할 핵심 수치를 배치했다.
- 일봉/주봉/월봉/분봉 차트를 붙이고 0B 실시간 현재가를 차트에 반영한다.
- KRX/NXT 구분, 장마감 후 스냅샷, 기준가 덮어쓰기 금지, 캐시 복원 규칙을 정리했다.

아직 세부 수치 검증과 TR 보정은 남아 있지만, 프로그램의 뼈대는 완성 단계에 들어왔다. 이후 개발은 "MTS와 수치 일치"를 최우선 기준으로 검증하고 보정한다.

## 핵심 원칙

- 기준가는 항상 KRX 전일종가다.
- 기준가는 NXT 종가, 현재가, LastPrice, 호가, 차트 종가, 캐시의 임의 가격으로 덮어쓰면 안 된다.
- NXT 종목을 보고 있어도 색상 판단과 등락률 기준은 KRX 전일종가다.
- 화면의 가격 색상은 기준가 초과 빨강, 기준가 미만 파랑, 기준가 동일 흰색이다.
- 조건검색식은 주기적 리로드가 아니라 WebSocket 조건검색 실시간 추적으로 편입과 이탈을 받는다.
- 한번 들어온 종목의 기본정보는 캐시에 남긴다. 이탈했다가 재편입되면 REST를 다시 긁지 않고 캐시로 복원하고 0B만 다시 맞춘다.
- 종목 선택이 바뀌면 이전 선택의 늦은 응답은 화면에 반영하지 않는다.
- 빈 응답이나 신뢰할 수 없는 값으로 기존의 유효한 화면 값을 지우지 않는다.

## NXT 종목의 표시 기준

MTS와 맞춘 중요한 규칙이다.

2026-05-29 기준, 삼성에스디에스 등 NXT 가능 종목을 MTS SOR ON 화면과 대조해 거의 일치하는 것으로 확인했다. 이 항목은 기준수치 확정 규칙으로 본다.

- 기준가: KRX 전일종가
- 색상 기준: KRX 전일종가
- 등락률 기준: KRX 전일종가 대비
- NXT 시간대의 시가/고가/저가/현재가: NXT 시세
- NXT 시간대의 호가/잔량: NXT 호가
- NXT 시간대의 거래량/거래대금: MTS SOR ON 기준에 맞춰 `_AL` 통합 일별거래상세를 우선 표시

즉, NXT 종목이라고 해서 기준가까지 NXT 20시 종가로 바꾸면 안 된다. 반대로 NXT 시간대에 보고 있는 시가/고가/저가/현재가를 KRX 값으로 강제로 되돌려도 MTS와 어긋난다.

화면 예:

- `기준가`: KRX 전일종가
- `시가`, `고가`, `저가`: 현재 선택 시장이 NXT이면 NXT 값
- 가격 글자색: 위 시가/고가/저가 값이 KRX 전일종가보다 높은지 낮은지로 판단

구현 기준:

- `GetStockStatusMetricsByGuideAsync(code, useNxtMarket: true)`는 `_NX` 코드로 OHLC/현재가 계열을 조회한다.
- `InfoBasePriceText`는 `StockStatusMetrics.BasePriceText`를 쓰지 않고, 선택 시작부에서 잠근 `_krxPrevClosePrice`만 표시한다.
- `ApplySelectedPriceInfoColors`는 NXT OHLC 가격을 KRX 전일종가와 비교해서 색상을 정한다.
- NXT 장중에는 KRX 종가 스냅샷을 적용하지 않는다. 장중 스냅샷이 들어오면 NXT 시/고/저/현재가를 KRX 값으로 덮어 MTS SOR ON 화면과 어긋난다.
- NXT 장중/고정 시간의 거래량, 거래대금, 전일거래량비율은 `_AL` 통합 일별거래상세로 보정하고, 이후 NXT 0B 단일 거래량이 이 값을 덮지 않도록 유지한다.
- NXT/SOR 표시 중 NXT 조회값이 비어도 KRX 시가/고가/저가/종가/호가/체결량으로 재조회해 채우지 않는다. KRX에서 가져와도 되는 값은 기준가로 잠근 전일종가뿐이다.

확정 상태:

- MTS SOR ON 기준으로 `기준가`, `시가`, `고가`, `저가`, `종가`, `거래량`, `거래대금`, 호가 색상 기준이 실사용 가능한 수준으로 맞았다.
- 약간의 차이는 REST 응답 시점, MTS 갱신 시점, NXT/통합 집계 반영 지연으로 보고 항목별 미세 보정 대상으로 둔다.
- 새 기능에서 이 규칙을 건드릴 때는 먼저 MTS 대조 로그를 남기고 수정한다.

## 수치 소유권 꼬리표 원칙

화면의 각 수치는 전담 데이터 출처를 가진다. 다른 TR이나 실시간 이벤트가 늦게 도착했다는 이유로 전담자가 아닌 값이 칸을 덮으면 안 된다.

이 원칙은 프로그램 칸에서 확인되었다. `ka90008`로 프로그램 순매수금액을 정상 표시했지만, 이후 `0B` 실시간 체결 갱신이 `_currentStatusMetrics`를 다시 그리면서 프로그램 값을 `-`로 덮었다. `0B`에는 프로그램매매 값이 없으므로 이 칸을 갱신할 권한이 없다.

소유권 예:

- 기준가: KRX 전일종가. NXT, `_AL`, `0B`가 덮을 수 없다.
- 시가/고가/저가/현재가: 현재 선택 시장의 시세 TR과 해당 시장 실시간.
- 거래량/거래대금: `_AL` 통합 일별거래상세 우선. `0B`는 보조로만 사용한다.
- 전일거래량대비: `_AL` 통합 일별거래상세와 전일 거래량.
- 매수/매도 체결량: 체결 요약 TR이 초기값을 소유하고, `0B`는 증가분 보조.
- 호가: `0D`와 호가 REST 스냅샷.
- 최근 체결: `0B`.
- 프로그램: `ka90008`. `0B`가 공란으로 덮을 수 없다.

구현 기준:

- 수치를 화면에만 쓰지 말고 해당 수치의 상태 객체에도 저장한다.
- 실시간 갱신은 자기 소유 칸만 바꾼다.
- 새 칸을 추가할 때는 먼저 "owner TR/WS", "보조 소스", "덮기 금지 소스"를 정한다.
- `ka90008` 프로그램 값은 `_currentStatusMetrics`에도 저장해 `0B` 실시간 재표시가 공란으로 덮지 못하게 한다.

## 0s 장운영구분

용도:

- KRX/NXT 사용 여부 판단
- NXT 장후 등록 여부 판단
- KRX 장마감 후 REST 스냅샷 보강 여부 판단

등록:

```json
{
  "trnm": "REG",
  "grp_no": "899",
  "refresh": "1",
  "data": [
    {
      "item": [""],
      "type": ["0s"]
    }
  ]
}
```

주요 필드:

- `215`: 장운영구분 코드
- `214`: 예상잔여시간 보조값
- `20`: 이벤트 시각

NXT 열림 판단:

- `P`, `R`, `T`, `U`

NXT 닫힘 판단:

- `Q`, `S`, `V`

초기 선조회가 시간초과되면 KRX 우선으로 시작하고, 이후 0s 이벤트가 오면 상태를 갱신한다.

## 0B 주식체결

등록:

```json
{
  "trnm": "REG",
  "grp_no": "900",
  "refresh": "1",
  "data": [
    {
      "item": ["005930"],
      "type": ["0B"]
    }
  ]
}
```

NXT 장후 또는 NXT 가능 시간에는 NXT 가능 종목을 `{종목코드}_NX`로 추가 등록한다.

주요 필드:

- `10`: 현재가
- `11`: 전일대비
- `12`: 등락률
- `13`: 누적거래량
- `15`: 체결량
- `20`: 체결시간
- `1030`: 매도체결량 누계
- `1031`: 매수체결량 누계
- `1315`: 매도 단건 체결량 보조
- `1316`: 매수 단건 체결량 보조

왼쪽 목록 반영:

- 현재가
- 누적거래량
- 등락률
- 가격 색상

최근 체결 반영:

- 체결가격
- 체결량
- 시장가 매수 체결량은 빨강
- 시장가 매도 체결량은 파랑

체결량 누계:

- `1030/1031` 누계가 있으면 누계 기준으로 표시한다.
- 누계가 없으면 `1315/1316` 또는 `15`의 부호와 직전 틱 가격으로 보조 판단한다.
- 장마감 후 REST 체결 합계가 신뢰되지 않으면 매수/매도체결량은 공란 처리한다.

차트 반영:

- 선택 종목의 0B가 들어오면 현재 차트의 마지막 봉을 갱신한다.
- 일봉/주봉/月봉은 현재 날짜, 주, 월 버킷 기준으로 마지막 봉을 갱신한다.
- 분봉은 `20` 체결시간을 분 단위 버킷으로 묶어 마지막 봉을 갱신한다.
- 부분 확대 중에도 현재가 가로선과 축 라벨은 0B 현재가를 따라 움직인다.

## 0D/0H 호가

등록 대상:

- 현재 선택 종목 1개
- NXT 사용 중이면 `{종목코드}_NX`
- 아니면 KRX 6자리 코드

등록:

```json
{
  "trnm": "REG",
  "grp_no": "901",
  "refresh": "1",
  "data": [
    {
      "item": ["005930"],
      "type": ["0D", "0H"]
    }
  ]
}
```

0D 주요 필드:

- 매도호가: `41`~`50`
- 매도잔량: `61`~`70`
- 매수호가: `51`~`60`
- 매수잔량: `71`~`80`

0H 주요 필드:

- `10`: 현재가
- `12`: 등락률

호가 화면 규칙:

- 20호가 구조다.
- 가격은 가운데 가격 줄에 이어서 표시한다.
- 매도호가는 위쪽, 매수호가는 아래쪽이다.
- 매도잔량은 오른쪽 정렬한다.
- 매수잔량은 왼쪽 정렬한다.
- 중앙 현재가 박스는 제거했다.
- 모든 호가 가격 옆에 기준가 대비 등락률을 표시한다.
- 등락률은 소수 둘째 자리까지만 자르고 반올림하지 않는다.
- 가격 색상과 등락률 색상은 KRX 전일종가 기준이다.
- 현재가와 같은 호가 가격 칸은 어두운 청록 배경과 하늘색 1px 테두리로 표시한다.
- 현재가 강조 칸은 글자 흔들림을 막기 위해 가운데 호가 컬럼 폭을 102px로 유지한다.

장마감 후 보강:

- KRX 15:30 이후 KRX 종목은 REST 최종 스냅샷으로 호가와 현재가를 채운다.
- NXT 가능 종목은 NXT 20:00 최종 상황을 다음날 07:00 전까지 고정값처럼 본다.

REST 스냅샷:

- `ka10004`: 주식호가
- `ka10007`: 시세표성정보

- 적으면 파랑
- 보합 또는 데이터 부족은 흰색/공란

NXT/KRX:

- KRX/NXT 차트 캐시는 서로 대체하지 않는다.

- 5: 핑크
- 10: 블루
- 20: 노랑
- 60: 그린
- 분봉 추가:
  - 240
  - 480

- 마우스 드래그로 구간 선택
- 더블클릭으로 전체 복귀
- 휠 확대/축소는 사용하지 않는다.
- 추가 과거 조회는 아직 하지 않는다.

부분 확대 시 이평선:

- 보이는 구간만으로 이평선을 다시 계산하지 않는다.
- 전체 로딩된 봉 기준으로 이평선을 계산하고, 화면에는 선택 구간만 그린다.
- 그래서 60/240/480선이 확대 구간 초반에서 불필요하게 잘리지 않아야 한다.

실시간 봉 갱신:

- 0B 현재가로 마지막 봉을 갱신한다.
- 같은 버킷이면 마지막 봉의 종가/고가/저가/거래량만 수정한다.
- 새 버킷이면 새 봉을 추가한다.
- 마지막 봉만 갱신 가능한 경우 전체 차트 재그리기를 피한다.
- 축 범위가 바뀌어야 하거나 새 봉이 생기면 전체 재그리기를 수행한다.

성능 로그:

- 예: `차트 전체그리기(3분봉 / 캐시): 700봉 / 160ms`
- 이 시간은 화면 그리기 시간이다.
- REST 데이터 조회 시간은 포함하지 않는다.

## REST 호출 제한 정책

목적:

- 키움 REST 429 방지

현재 정책:

- 최대 5 calls/sec 기준
- 지속 모드 간격 200ms
- 동시 게이트 5
- 429 또는 제한 return_code 수신 시 cooldown

관련 구현:

- `WaitRestRateLimitAsync`
- `SendKiwoomRestAsync`

주의:

- 빠른 종목 연속 선택에서 이미 서버로 전송된 REST 요청은 취소해도 서버에서는 처리될 수 있다.
- 따라서 UI 반영은 선택 버전으로 막고, REST 요청 자체는 호출 제한을 지켜야 한다.

## 완료된 주요 기능

- 조건검색식 01번 초기 조회
- 조건검색 실시간 편입/이탈 추적
- 편입 종목 캐시 복원
- 관심종목 0B 실시간 등록
- 선택 종목 0D/0H 호가 등록
- KRX/NXT 장상태 판단
- KRX 전일종가 기준가 잠금
- 호가 가격/등락률 색상 기준가 적용
- 장마감 후 REST 스냅샷
- 일봉/주봉/월봉/분봉 차트
- 분봉 1/3/5/10/15/30/60/120
- 차트 드래그 부분 확대
- 이평선 전체봉 기준 계산
- 부분 확대 중 현재가 가로선 실시간 이동
- 뉴스/공시 표시
- REST 429 방어
- 시작 로딩 오버레이

## 개발 시 주의할 점

- 기준가를 KRX 전일종가 외 값으로 바꾸지 말 것.
- NXT 현재가와 KRX 기준가를 혼동하지 말 것.
- 조건검색식은 주기 리로드가 아니라 실시간 이벤트 기반이다.
- 편입/이탈했던 종목은 캐시로 처리하고 REST 재조회하지 않는 흐름을 유지할 것.
- 종목 선택 작업은 항상 선택 버전과 취소 토큰을 고려할 것.
- 빈 응답으로 기존 화면 값을 지우지 말 것.
- 새 화면 항목을 추가하면 이 문서에 데이터 출처, 계산 기준, 갱신 방식을 같이 추가할 것.
