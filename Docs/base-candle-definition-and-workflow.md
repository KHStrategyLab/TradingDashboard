# 기준봉 정의와 적용 워크플로우

이 문서는 TradingDashboard에서 기준봉을 어떻게 정의하고, 화면 목록과 전략 진행률에 어떻게 적용하는지 고정한다.

## 1. 기준봉 정의

TradingDashboard에서 그냥 `기준봉`이라고 쓰면 KRX 정규장 일봉 기준봉을 뜻한다.

- 기준 시장: KRX 정규장
- 기준 차트: 일봉
- 기준 역할: 이후 눌림, 회복, 돌파 판단의 중심 가격대
- 금지사항: NXT 종가, SOR 통합값, 장중 1분봉 장대양봉이 기존 `BaseCandle*`, `BaseHalfPrice`를 덮어쓰면 안 된다.

기준봉은 이후 매수 신호를 바로 만드는 값이 아니다. 기준 가격대를 제공하고, 전략이 그 기준 위에서 눌림과 회복을 추적하게 한다.

## 2. 기준봉 게이트

조건식에서 들어온 종목을 모두 화면에 올리면 볼 종목이 너무 많아진다. 그래서 조건식 편입 뒤, 화면 표시 전에 `기준봉 게이트`를 둔다.

게이트 조건:

```text
0봉전 기준 최근 10거래일 이내
거래대금 1,000억 이상
AND
전일대비등락률 20% 이상
```

거래대금 단위:

```text
1,000억 = 100,000,000,000원
게이트 거래대금 = Max(ka10081 거래대금 후보, 종가 * 거래량)
```

`ka10081`의 `trde_prica` 계열 값은 응답에 따라 백만원 단위 또는 축약 단위처럼 들어올 수 있으므로, 게이트에서는 반드시 `종가 * 거래량`으로 원 단위 추정 거래대금을 검산한다.

검사 범위:

```text
0봉전, 1봉전, 2봉전 ... 9봉전
```

통과 처리:

- 왼쪽 목록에 표시한다.
- 0B 등록 대상이 된다.
- `GateBaseCandle*` 필드에 통과 봉 정보를 저장한다.
- 목록 배지에 `D+0`, `D+3`처럼 표시한다.
- `D+3`은 기준봉 발생 후 3거래일째 추적 중이라는 뜻이다.
- `D+` 숫자는 파일에 저장하지 않는다. 최근 일봉 검사 결과로 메모리에서 계산한다.
- KRX/NXT 같은 시장 표시는 기준봉 배지나 툴팁에 붙이지 않는다.
- 시장구분과 NXT 가능 여부는 종목 자체 시장/`NX` 표시가 담당한다.
- 시장 근거는 `GateBaseCandleMarket`에 저장만 한다.

실패 처리:

- 화면에 표시하지 않는다.
- 실패 결과를 캐시에 저장하지 않는다.
- 이후 조건식에 다시 편입되면 다시 검사한다.

## 3. 게이트 기준봉과 전략 기준봉 구분

`GateBaseCandle*`는 목록 진입용 문이다.

`BaseCandle*`, `BaseHalfPrice`는 전략 기준봉이다.

두 필드는 역할이 다르므로 서로 덮어쓰면 안 된다.

```text
GateBaseCandle* = 볼 종목인지 거르는 입장권
BaseCandle*     = 전략이 눌림/회복/돌파를 계산하는 기준 가격대
```

## 4. KRX / NXT / SOR 적용

KRX 종목:

- KRX 일봉으로 게이트를 검사한다.
- KRX 일봉 통과 결과만 저장한다.

NXT 지원 종목:

- KRX 일봉을 먼저 검사한다.
- NXT 시간대 또는 NXT 고정 시간대라면 NXT 일봉을 보조 게이트로 볼 수 있다.
- 기준가 색상과 등락률 비교 기준은 여전히 KRX 전일종가다.
- NXT 게이트 통과 결과는 `GateBaseCandleMarket = NXT`로 표시한다.

SOR:

- SOR는 주문 라우팅 방식이다.
- 게이트 판정 자체와 주문 종목코드에 `_NX`, `_AL` suffix를 섞지 않는다.
- SOR/NXT 흐름은 기준봉을 대체하지 않고 보조 판단으로만 둔다.

## 5. 현재 구현 흐름

초기 조건식 조회:

```text
CNSRLST
-> CNSRREQ(search_type=1)
-> 최초 조건검색 결과 수신
-> 종목 기본정보 보강
-> 기준봉 게이트 검사
-> 통과 종목만 왼쪽 목록 표시
-> 통과 종목만 캐시 저장
-> 통과 종목만 0B 등록
```

실시간 편입:

```text
조건검색 REAL 편입
-> 캐시에 오늘 게이트 통과 결과가 있으면 즉시 복원
-> 없으면 기본정보 조회
-> 별도 일봉 게이트 재조회 없이 NEW로 목록 표시
-> 0B 등록
-> 장중 신규 편입은 파일 저장 없음
```

장중 신규 편입:

- 조건검색 실시간 편입으로 새로 들어온 종목은 기준봉 게이트를 통과하지 못해도 화면에 표시한다.
- 화면에는 파란 `NEW` 배지를 붙인다.
- 파일에는 저장하지 않는다.
- 0B 등록 대상에는 포함한다.
- 신규 편입 확인을 위해 별도 폴링을 돌리지 않는다. 이미 열린 조건검색 WebSocket 이벤트만 처리한다.
- 장중 NEW 편입은 일봉 게이트를 다시 조회하지 않는다. 오늘 캐시에 통과 이력이 있는 종목만 `D+`로 복원하고, 순수 신규 편입은 `NEW`로만 표시한다.
- 편입 이벤트가 몰려도 기본정보/뉴스/공시 보강은 한 번에 하나씩 처리하며, 같은 종목이 이미 처리 대기 중이면 중복 작업을 만들지 않는다.
- 장중 재실행 시 이전 실행의 `NEW` 종목은 복원하지 않는다. 초기 조건식 조회에서 게이트를 통과하지 못하면 컷되고, 이후 실시간 편입 이벤트가 다시 오면 `NEW`로 표시한다.
- 매수 가능 여부, 알림만 처리할지, 별도 전략으로 태울지는 전략실이 판단한다.
- 목록 레이어는 장중 신규 편입 종목을 살려서 전략실로 넘기는 역할까지만 한다.

목록 거래대금 신호:

- 왼쪽 목록에는 당일 거래대금을 `xxx억` 형태로 표시한다.
- 200억 이상은 빨강, 200억 미만은 파랑이다.
- 이 신호는 현재 화면 판단용이며, 기준봉 게이트의 1,000억 조건과 별개다.

기준봉 게이트 통과 표시:

- 기준봉 게이트 통과 종목은 빨간 `D+숫자` 배지를 붙인다.
- 당일 기준봉 게이트 통과는 `D+0`이며 더 강한 빨강으로 표시한다.
- `NEW`와 `D+0`이 함께 붙을 수 있다. 이 경우 장중 신규 편입이면서 당일 기준봉 게이트도 통과한 종목이다.

## 6. 진행률 연결

전략 진행률에서 기준봉 게이트는 초기 문이다.

예시:

```text
0%   조건식 편입
10%  기준봉 게이트 통과
30%  전략 기준봉 확인
45%  눌림 확인
60%  반응/돌파 대기
70%  매수 체결 완료
100% 매도 완료
```

70%는 매수 완료 기준이고, 100%는 매도까지 끝난 전략 종료 기준이다.

## 7. 개발 주의점

- 기준봉 게이트 실패 종목을 캐시에 실패 상태로 저장하지 않는다.
- 게이트 통과 결과만 저장한다.
- 게이트의 `D+` 숫자는 저장하지 않는다.
- 게이트 조건을 바꿀 때는 목록 표시 종목 수와 0B 등록 종목 수가 함께 바뀐다.
- 게이트가 강해질수록 종목 수는 줄고, 놓치는 종목도 늘어난다.
- 게이트가 약해질수록 화면과 0B 등록 대상이 늘어나므로 속도와 제한을 다시 확인해야 한다.
- 용어가 헷갈리면 `Docs/trading-dashboard-glossary.md`의 `기준봉`, `기준봉 게이트`, `게이트 기준봉`을 먼저 따른다.
