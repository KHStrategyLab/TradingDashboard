# TradingDashboard 계산식 개발 기록

작성일: 2026-05-29

이 문서는 TradingDashboard 화면에 표시하는 주요 수치의 계산 기준을 남긴 기록이다. 특히 KRX/NXT/SOR 통합 데이터, 0봉/1봉 개념, MTS와 맞춘 표시 기준을 다시 헷갈리지 않기 위해 작성했다.

## 기본 원칙

- 주식 데이터에서 전일, 전거래일은 날짜 산술이 아니라 봉 기준으로 본다.
- 0봉은 최신 거래봉, 1봉은 바로 이전 거래봉이다.
- 오늘 거래량/전일동시/프로그램매매처럼 KRX와 NXT가 섞이는 값은 가능한 한 `_AL` 통합 코드를 우선 사용한다.
- 기준가와 호가 색상 기준은 KRX 전일종가를 고정 기준으로 사용한다.
- 장마감 후 매수체결량/매도체결량 총계는 신뢰 가능한 총량 TR이 없어 공란 처리한다.

## 기준가

목적:

- 오늘 가격의 상승/하락 색상 기준
- 호가창 등락률 색상 기준

사용 TR:

- `ka10007` 시세표성정보요청
- 요청 종목코드: KRX 기본 코드

우선 필드:

- `pred_close_pric`
- fallback: `base_prc`, `std_prc`, `yday_prc`, 응답 내 전일종가 계열 값

계산:

```text
기준가 = KRX 전일종가
등락금액 = 현재가 - 기준가
등락률 = 등락금액 / 기준가 * 100
```

주의:

- NXT 종목이어도 기준가는 KRX 전일종가를 사용한다.
- 이후 다른 TR이 기준가를 덮어쓰지 않는다.

## 오늘 거래량

목적:

- 종목정보의 오늘 거래량 내부 기준
- 전일동시 비율 계산의 0봉 분자
- 회전율 계산의 분자

사용 TR:

- `ka10015` 일별거래상세요청
- 요청 종목코드: `{종목코드}_AL` 우선
- fallback: 현재 선택 시장 코드, 예: KRX 기본 코드 또는 `{종목코드}_NX`

우선 필드:

- `trde_qty`
- fallback: `tot_3`

계산:

```text
오늘 거래량 = ka10015(_AL).daly_trde_dtl[0].trde_qty
장구분 합계 = 장전거래량 + 장중거래량 + 장후거래량
```

검증 예:

```text
삼화콘덴서 총 13,643,274
장전 29 + 장중 13,639,338 + 장후 3,907 = 13,643,274
MTS 표기 13,643K와 일치
```

## 오늘 거래대금

사용 TR:

- `ka10015` 일별거래상세요청
- 요청 종목코드: `{종목코드}_AL` 우선

우선 필드:

- `trde_prica`
- 단위: 백만원

fallback:

- `bf_mkrt_trde_prica + opmr_trde_prica + af_mkrt_trde_prica`

계산:

```text
오늘 거래대금(백만원) = trde_prica
fallback 거래대금(백만원) = 장전거래대금 + 장중거래대금 + 장후거래대금
```

표시:

```text
100백만원 이상이면 억 단위로 표시
그 미만은 백만 단위로 표시
```

## 전일동시

MTS 표시명:

- 전일동시

의미:

- 오늘 0봉 거래량을 전거래일 1봉 거래량과 비교한 비율

사용 TR:

- `ka10015` 일별거래상세요청
- 요청 종목코드: `{종목코드}_AL`

계산:

```text
오늘 거래량 = ka10015(_AL) 최신 행, 즉 0봉 trde_qty
전일 거래량 = ka10015(_AL) 바로 이전 행, 즉 1봉 trde_qty
전일동시 = 오늘 거래량 / 전일 거래량 * 100
```

표시:

```text
전일동시 = truncate(비율, 소수 둘째 자리) + "%"
오늘 거래량 > 전일 거래량이면 빨강
오늘 거래량 < 전일 거래량이면 파랑
같으면 흰색
```

주의:

- 날짜를 직접 계산하지 않는다.
- 정렬된 일별 데이터에서 0봉/1봉으로 판단한다.
- 오늘 거래량이 `_AL`이면 전일 거래량도 `_AL`이어야 한다.

## 유통주식수

사용 TR:

- 우선: `ka10007`
- 보조: `ka10095`, `ka10100`

우선 필드:

- `flo_stkcnt`

fallback:

- `listCount`
- `stkcnt`
- `lst_stk_cnt`
- `list_stkcnt`

단위 보정:

```text
원본 값이 1,000,000 미만이면 천주 단위로 보고 * 1000
원본 값이 1,000,000 이상이면 1주 단위로 보고 그대로 사용
```

예:

```text
10,395 -> 10,395,000주
10,395,000 -> 10,395,000주
```

## 회전율

MTS 기준에 맞춘 계산:

```text
회전율 = 오늘 거래량 / 유통주식수 * 100
```

분자:

- 오늘 거래량
- `ka10015(_AL)` 0봉 `trde_qty`

분모:

- 유통주식수
- `ka10007.flo_stkcnt` 우선

표시:

```text
회전율 = truncate(비율, 소수 둘째 자리) + "%"
```

검증 예:

```text
삼화콘덴서
오늘 거래량 8,193,246 기준일 때
유통주식수 약 6,242,473
8,193,246 / 6,242,473 * 100 = 131.25%
```

주의:

- 상장주식수 기준으로 나누면 MTS 회전율과 맞지 않는다.
- 시총 계산에는 전체 상장주식수를 사용하고, 회전율에는 유통주식수를 사용한다.

## 시가총액

사용 값:

- 현재가
- 전체 상장주식수

계산:

```text
시가총액 = 현재가 * 전체 상장주식수
```

전체 상장주식수 필드:

- `ka10100.listCount`
- fallback: `stkcnt`, `lst_stk_cnt`, `list_stkcnt`

주의:

- 시총은 유통주식수가 아니라 전체 상장주식수 기준이다.

## 프로그램

MTS와 맞춘 표시 기준:

- 수량이 아니라 프로그램 순매수금액을 표시한다.

사용 TR:

- `ka90008` 종목시간별프로그램매매추이요청
- 요청 종목코드: `{종목코드}_AL`
- 요청 날짜: 0봉 일자
- 요청 파라미터: `amt_qty_tp = "2"`

표시 필드:

- `prm_netprps_amt`

계산:

```text
프로그램 = prm_netprps_amt
```

fallback:

```text
프로그램 = prm_buy_amt - prm_sell_amt
```

표시 색상:

```text
양수면 빨강
음수면 파랑
0 또는 없음이면 흰색 또는 "-"
```

주의:

- 응답에 `--869`처럼 들어오는 값은 `-869`로 정규화해서 파싱한다.
- MTS의 프로그램 표시가 금액 기준이면 `prm_netprps_qty`가 아니라 `prm_netprps_amt`를 사용한다.

## 매수체결 / 매도체결 / 매수비율

장중:

- WebSocket `0B` 또는 REST `ka10003` 체결 정보를 통해 누적 관리할 수 있다.
- 체결량 부호 또는 방향 필드로 매수/매도 방향을 판단한다.

장마감 후:

- 총 매수체결량/총 매도체결량을 신뢰 가능하게 복원하는 단일 TR이 없어 공란 처리한다.

매수비율:

```text
매수비율 = 매수체결량 / (매수체결량 + 매도체결량) * 100
```

주의:

- 장마감 후에는 매수체결/매도체결/매수비율을 표시하지 않는다.
- 최근 체결 리스트 일부를 합산해 총량처럼 보이면 안 된다.

## 호가잔량 합계와 차이

사용 데이터:

- 호가창 10호가 매도잔량
- 호가창 10호가 매수잔량

계산:

```text
총매도잔량 = 매도 1호가부터 10호가 잔량 합
총매수잔량 = 매수 1호가부터 10호가 잔량 합
차이 = 총매도잔량 - 총매수잔량
```

표시:

```text
총매도잔량: 매도잔량 세로줄 아래
총매수잔량: 매수잔량 세로줄 아래
차이: 호가창 아래 가운데
차이가 양수면 파랑
차이가 음수면 빨강
부호는 표시하지 않고 절대값만 표시
```

## NXT/KRX 통합 주의

- 차트는 NXT 가능 종목이면 NXT 차트 우선 표시한다.
- 오늘 거래량, 거래대금, 전일동시, 프로그램은 MTS와 맞추기 위해 `_AL` 통합값을 우선 사용한다.
- 호가 색상 기준과 기준가는 KRX 전일종가를 고정 사용한다.
- NXT 응답이 비어 있을 때 기존 유효 호가를 공백으로 덮지 않는다.
