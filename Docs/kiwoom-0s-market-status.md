# Kiwoom 0s 장운영구분 개발 기록

작성일: 2026-05-29

이 문서는 TradingDashboard에서 Kiwoom WebSocket 실시간 타입 `0s`(장시작시간/장운영구분)를 사용하면서 정리한 운영 규칙과 구현 판단을 남긴 기록이다. `0s`는 KRX/NXT 데이터 선택, 장마감 스냅샷, 장전/장중/장후 판단의 출발점이므로 앱 시작 시 가장 먼저 확인한다.

## 핵심 결론

- 앱 시작 후 조건식 조회보다 먼저 `0s`를 등록한다.
- `0s`에서 가장 중요한 값은 `215` 장운영구분이다.
- `214` 장시작예상잔여시간은 주로 KRX 기준 보조값으로 보고, NXT 판단의 주 기준으로 쓰지 않는다.
- `0s` 초기 수신이 항상 즉시 오는 것은 아니므로 짧은 타임아웃을 둔다.
- 타임아웃이면 상태 미확인으로 기록하고, 보수적으로 KRX 우선으로 시작한다.
- 이후 `0s` 이벤트가 들어오면 그 값으로 장상태와 NXT 사용 여부를 갱신한다.
- 휴장일, 서버 지연, 미수신 가능성이 있으므로 `0s` 단독 판단이 아니라 시간대와 실제 TR 응답으로 보완한다.

## WebSocket 등록 규격

로그인 후 아래 REG 패킷으로 등록한다.

```json
{
  "trnm": "REG",
  "grp_no": "1",
  "refresh": "1",
  "data": [
    {
      "item": [""],
      "type": ["0s"]
    }
  ]
}
```

응답은 `REAL` 메시지의 `values` 안에서 확인한다.

주요 필드:

| 필드 | 의미 | 사용 방침 |
| --- | --- | --- |
| `215` | 장운영구분 | KRX/NXT 및 장상태 판단의 주 기준 |
| `214` | 장시작예상잔여시간 | KRX 기준 보조값으로만 사용 |
| `20` | 체결시간 또는 이벤트 시각 계열 값 | 로그/검증용 보조값 |

## 215 장운영구분 사용 방침

`215`는 현재 장 상태를 나타내는 값이면서 상태 전환 이벤트로도 들어온다. 프로그램을 장중에 켜도 현재 상태가 한 번 내려올 가능성이 높지만, 항상 보장된다고 보지는 않는다.

운영 판단:

- NXT 장이 이미 열린 상태에서 프로그램을 켜면 `P` 같은 NXT 상태값이 내려올 수 있다.
- `P/Q/R/S/T/U/V` 계열은 NXT 프리/메인/애프터 관련 상태 판단의 중심으로 사용한다.
- 장중이면 시작 직후든 중간이든 해당 상태 코드가 유지되어 내려올 가능성이 높다고 본다.
- 휴장일에는 별도 휴장 전용 값이 명확히 오지 않거나 응답 자체가 지연될 수 있다.

## 214 잔여시간 취급

`214`는 장시작예상잔여시간이지만 NXT 프리/메인/애프터 각각의 정확한 잔여시간으로 신뢰하지 않는다.

운영 원칙:

- KRX 정규장 보조값으로만 사용한다.
- NXT 거래 가능 여부는 `215` 상태값과 실제 NXT 시세/TR 응답으로 판단한다.
- NXT 장전 시간대 거래 가능 여부는 `0s`만으로 명확하지 않을 수 있다.

## 앱 시작 순서

현재 권장 순서:

1. 앱 시작
2. `0s` 장운영구분 선조회 시작
3. WebSocket 로그인
4. `0s` 등록
5. 1~2초 정도 초기 `215` 수신 대기
6. 수신 성공 시 장상태 반영
7. 수신 실패 시 `장상태 미확인(KRX 우선)`으로 기록
8. 조건식 조회 시작
9. 조건식 종목 목록 구성
10. 종목별 NXT 가능 여부, 시장구분, 기준가 캐시 반영
11. 종목 선택 시 장상태와 종목 속성에 맞는 KRX/NXT 데이터 사용

실제 로그 예:

```text
[03:16:34] 앱 시작
[03:16:34] 0s 장운영구분 선조회 시작
[03:16:35] 0s 장운영구분 등록
[03:16:36] 0s 장운영구분 선조회 시간초과, 장상태 미확인(KRX 우선)
[03:16:36] 키움 조건식(01) 조회 시작
```

## KRX/NXT 데이터 선택 원칙

종목 선택 후 데이터별 기준을 분리한다.

- 기준가: 항상 KRX 전일종가를 기준으로 색상/등락 계산을 맞춘다.
- 호가: NXT 가능 종목이고 NXT 운영/마감 후 보존 시간대라면 NXT 호가를 우선 사용한다.
- 차트: NXT 가능 종목은 NXT 차트를 우선 표시한다. 특히 장마감 후부터 다음날 07:00 전까지는 NXT 표시 원칙을 유지한다.
- 프로그램매매: MTS와 동일 표시를 위해 `ka90008 + 종목코드_AL + 0봉 dt + prm_netprps_amt`를 사용한다.
- 매수/매도 체결량 누계: 장마감 후에는 신뢰 가능한 총 매수/매도 체결량 TR이 없어 공란 처리한다.

## 장마감 후 스냅샷

장마감 후에는 실시간 마지막 수신값이 비거나 오래된 값일 수 있으므로 REST 스냅샷을 사용한다.

KRX:

- KRX 정규장 종료 후 KRX 최종 스냅샷 사용
- KRX 종목은 KRX 스냅샷 유지

NXT:

- NXT 가능 종목은 NXT 20시 최종 스냅샷 사용
- NXT 스냅샷이 비어 있으면 기존 유효 데이터를 공백으로 덮지 않는다.
- 장마감 후 매수/매도 체결량은 공란 처리한다.

## 타임아웃과 보완 판단

`0s`는 반드시 즉시 온다고 가정하지 않는다.

방침:

- 초기 대기 시간 내 미수신이면 KRX 우선으로 시작한다.
- 이후 `0s`가 들어오면 NXT/KRX 모드를 재판단한다.
- 휴장일 또는 지연 수신 가능성은 로그에 남긴다.
- 실제 호가/차트/시세 TR 응답이 비어 있으면 기존 유효 데이터 보존 정책을 우선한다.

## 주의한 시행착오

- `0s`보다 조건식/종목 TR을 먼저 돌리면 NXT/KRX 판단이 늦어져 화면 데이터가 꼬일 수 있다.
- 장마감 후 KRX 호가를 먼저 표시한 뒤 빈 NXT 응답으로 덮으면 호가창이 공백이 되는 문제가 있었다.
- 빈 NXT 응답은 기존 호가를 지우지 않는 방어가 필요하다.
- `214`만 보고 NXT 시간대를 판단하면 위험하다.
- 프로그램 시작 시 `0s`가 타임아웃되더라도 나중에 들어오는 이벤트를 계속 반영해야 한다.

## 향후 확인 질문

- `215` 코드 전체 표와 각 코드의 정확한 KRX/NXT 의미
- NXT 장전 시간대 거래 가능 상태가 `215`에서 별도로 구분되는지
