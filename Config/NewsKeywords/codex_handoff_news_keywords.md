# 뉴스 긍정 단어 사전 자료 묶음

## 목적

뉴스 제목과 본문에서 긍정/가치/기대/주의/부정 단어를 찾아서 뉴스판 옆에 태그처럼 보여주기 위한 자료다.

이 파일 묶음은 구현 코드가 아니다.  
Codex가 프로그램에 붙일 때 참고할 외부 사전 자료와 적용 원칙만 담았다.

## 최상위 태그

- 긍정
- 가치
- 기대
- 주의
- 부정

## 큰 방향

첫번째, 뉴스 단어는 C# 코드 안에 직접 박지 않는다.

두번째, `news_keywords.json` 파일에서 단어 사전을 불러온다.

세번째, 로드 실패 시 프로그램을 멈추지 않는다.

네번째, 긍정 단어가 있어도 주의/부정 단어가 같이 있으면 반드시 주의 태그도 같이 보여준다.

다섯번째, 이 기능은 매수 신호가 아니라 뉴스 해석 보조용이다.

## 파일 구성

`news_keywords.json`  
실제 단어 사전 자료다. 카테고리, 감정 분류, 강도, 태그, 단어 목록이 들어 있다.

`news_keywords_add_template.json`  
나중에 단어를 추가할 때 쓰는 빈 틀이다.

`codex_handoff_news_keywords.md`  
Codex에게 전달할 설명 파일이다.

## 적용 원칙

제목에서 잡힌 단어는 본문보다 강하게 본다.

본문에서만 잡힌 단어는 보조 태그로 본다.

긴 단어를 먼저 잡는다.  
예: `자사주소각`을 먼저 잡고, 그 다음 `자사주`를 잡는다.

충돌 단어를 조심한다.  
예: `투자`는 긍정일 수 있지만 `투자주의`, `투자경고`는 주의/부정이다.  
예: `상승`은 긍정일 수 있지만 `금리상승`, `원가상승`은 종목에 부정일 수 있다.

`기대감`, `가능성`, `전망`, `논의`, `관측`은 단독으로는 약한 긍정이다.  
구체 재료와 같이 잡히면 중간 긍정 이상으로 올릴 수 있다.

## Codex 전달 문구

뉴스 긍정 단어 사전 확장 작업은 코드 하드코딩이 아니라 외부 JSON 파일 기반으로 진행한다.

초기 목적은 뉴스판에서 제목/본문에 포함된 단어를 태그처럼 보여주는 것이다.  
자동 매수 판단에 바로 연결하지 않는다.

`news_keywords.json`을 읽어서 카테고리별 단어를 로드하고, 제목/본문에서 매칭된 단어를 태그로 표시한다.

파일 로드가 실패해도 프로그램은 중단하지 않는다.  
기본 최소 사전 또는 이전 백업 사전을 사용하도록 설계한다.

긍정 단어가 있더라도 주의/부정 단어가 같이 잡히면 [주의] 또는 [부정] 태그를 함께 표시한다.

단어 추가는 앞으로 JSON 파일만 수정하는 방식으로 관리한다.
