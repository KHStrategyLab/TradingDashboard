# AGENTS.md

## 역할

이 저장소는 TradingDashboard / Kiwoom 기반 주식 매매 보조 프로그램이다.
Codex는 코드를 수정할 때 기존 정상 동작을 최대한 유지하고, 사용자가 확정한 매매 기준과 KRX/NXT 기준을 절대 섞지 않는다.

## 기본 작업 원칙

- 대규모 리팩토링을 먼저 시도하지 않는다.
- 정상 동작 중인 흐름을 깨지 않는 최소 수정부터 한다.
- UI 정리보다 데이터 정확도, 매매 기준, 주문 안전성을 우선한다.
- 변경 전후로 영향 범위를 먼저 확인한다.
- 확실하지 않은 구조는 추측으로 갈아엎지 말고, 기존 코드 흐름을 따라간다.
- 빌드 실패 상태로 작업을 끝내지 않는다.
- 작업 결과에는 변경 파일, 변경 이유, 빌드 결과를 남긴다.
- 소스/문서 파일은 `.editorconfig` 기준 UTF-8로 유지한다.
- 코드의 UI/로그 고정 문구는 가능하면 영어로 둔다. 한글은 문서, 사용자 데이터, API 응답 매칭 토큰처럼 필요한 곳에만 둔다.

## 문서화 원칙

- MD 문서는 계획서가 아니라 다음 개발자가 이어받을 기준서로 작성한다.
- 코드 수정이 성공하고 빌드/검증이 끝난 뒤에만 문서를 업데이트한다.
- 방향이 맞지 않거나 빌드가 실패하면 문서를 먼저 쓰지 말고 코드를 다시 고친다.
- 문서에는 실제 반영된 내용, 남은 주의점, 다음 작업 기준만 적는다.
- 용어가 헷갈릴 때는 `Docs/trading-dashboard-glossary.md`를 먼저 따른다.

## KRX / NXT 기준

- KRX와 NXT 데이터 기준을 섞지 않는다.
- NXT 종목의 화면 표시용 전일대비와 등락률은 KRX 전일종가 기준으로 계산/표시한다.
- NXT 종목의 시가, 고가, 저가, 종가와 장중 수치는 NXT 기준값을 사용한다.
- NXT 현재가, 거래량, 거래대금, 장중 실시간 수치는 ALL/NXT 포함 MTS 기준과 맞는지 우선 확인한다.
- NXT 화면/잔고 보정 작업 중 KRX 종목정보가 NXT 값을 덮어쓰지 않도록 주의한다.
- 같은 종목이라도 KRX 분봉과 NXT 분봉은 다를 수 있으므로 시장별 MinuteChartMarket, RealtimePriceMarket, DisplayMarket 기준을 보존한다.

## 주문 안전 원칙

- NXT 주문을 KRX 본장 주문과 동일하게 취급하지 않는다.
- NXT 주문은 시장가 주문을 기본값으로 사용하지 않는다.
- NXT 주문은 현재가 기준 ±1~3호가 지정가 주문을 기본 검토한다.
- NXT에서 지원하지 않는 주문 유형은 보내기 전에 차단한다.
- 주문 방식은 코드에서 임의로 단순화하지 않는다.
- 실주문 관련 변경은 특히 보수적으로 처리한다.

## 기준봉 / 후보 관리 원칙

- 현재 TradingDashboard는 `Kiwoom.ConditionSeq01` 설정값 하나를 화면 후보 목록과 전략 후보 seed로 같이 사용한다.
- `조건00` 같은 명칭은 기존 전략/문서의 전략명일 수 있으므로, 현재 앱의 키움 조건식 설정값으로 오해하지 않는다.
- 기준봉 OHLC와 BaseHalfPrice는 KRX 기준 기준봉 데이터를 사용한다.
- NXT 후보라도 기준봉 자체는 KRX 검색식 기준봉을 따른다.
- 기준봉 판단은 KRX 정규장 일봉을 우선하되, SOR/NXT 흐름은 당일 흐름 확인과 보조 판단으로 분리해 사용한다.
- 조건식 편입 종목은 화면 목록에 올리기 전에 0봉전 기준 최근 10거래일 기준봉 게이트를 통과해야 한다.
- 기준봉 게이트 기본값은 거래대금 1,000억 이상 AND 전일대비등락률 20% 이상이다.
- 게이트를 통과하지 못한 종목은 화면에 표시하지 않지만, 실패 결과를 캐시에 저장하지 않는다. 이후 조건식에 다시 편입되면 다시 검사한다.
- 단, 장중 조건검색 실시간으로 새로 편입된 종목은 하이패스 후보로 보고 `NEW` 배지만 붙여 표시한다.
- 장중 `NEW` 편입 여부 확인을 위해 별도 폴링을 만들지 않는다. `CNSRREQ(search_type=1)` WebSocket 이벤트만 처리한다.
- 장중 `NEW` 편입에서는 일봉 게이트를 다시 조회하지 않는다. 오늘 캐시에 게이트 통과 이력이 있는 종목만 `D+`로 복원한다.
- 장중 편입 이벤트의 REST 보강 작업은 중복 실행하지 않고 순차 처리한다.
- 게이트 통과 결과는 `GateBaseCandle*` 필드에 저장하며, 기존 KRX 전략 기준봉 `BaseCandle*`, `BaseHalfPrice`를 덮어쓰지 않는다.
- 기준봉 데이터 저장/로드 구조를 임의로 바꾸지 않는다.
- 후보 JSON 필드의 StrategyMarket, NxtEnabled, StrategyCode, StrategyGroup, MinuteChartMarket, RealtimePriceMarket, DisplayMarket 값을 보존한다.

## 매매 전략 핵심 흐름

- 기준봉 다음날 무조건 추격매수하지 않는다.
- 기준봉 이후 눌림과 회복을 기다리는 구조를 유지한다.
- 10분봉 MA60 하향 이탈 확인 후 재회복 흐름을 중요하게 본다.
- 기준봉 저가 위 유지 여부를 확인한다.
- 10분봉 MA10 눌림/접촉과 5분봉 20개 신고가 돌파 흐름을 매수 후보 판단에 활용한다.
- 현재 전략은 추격도 바닥매수도 아닌, 기준봉 이후 눌림-회복 중간 구간을 잡는 구조다.
- 전략 로직을 바꿀 때는 손익비, 기준가, 이평 신호, 거래대금 흐름을 함께 확인한다.

## 10분봉 / 5분봉 / MA 신호등

- 후보 그리드에는 10분봉 MA5, MA20, MA60 숫자와 신호등을 표시한다.
- 신호등 규칙:
  - 초록: MA5 > MA20 그리고 MA5 > MA60
  - 빨강: MA60 > MA5 그리고 MA60 > MA20
  - 노랑: 그 외 모든 경우
- 완전 정배열 MA5 > MA20 > MA60은 별도 강조 가능하지만, 기존 초록 규칙을 깨지 않는다.
- 신호등 갱신으로 행 순위가 바뀌거나 리셋되거나 깜빡이면 안 된다.
- 현재봉 포함 실시간 계산은 허용하되, 10분 정각 확정봉 교체 구조를 깨지 않는다.
- 재시작 또는 WS 재접속 후 초기 REST 분봉 로드 성공 전에는 매수 판단을 차단한다.

## 점수 / 게이트 / 비중

- 점수 체계는 A/B 2단계를 기본으로 한다.
- ScorePercent가 70점 이상이면 A, 70점 미만 또는 누락이면 B로 본다.
- A는 종목당 진입예산 100%, B는 50%를 기본으로 한다.
- 전체 계좌가 아니라 사용자가 설정한 자동매매 예산과 종목당 예산 안에서만 계산한다.
- 관찰 후보를 0%로 완전 배제하지 않는다.
- RiskGuard는 목표가, 손절가, 중복, 슬롯, 예산 체크 후 수량을 계산한다.

## 뉴스 / 공시 필터

- 뉴스와 공시는 매매 흐름을 주도하는 메인이 아니라 보조 재료 확인용이다.
- 매매 흐름은 키움 조건, 체결, 호가, 거래대금, 가격 움직임이 주도한다.
- 뉴스 알림은 키워드 필터를 통과한 것만 보낸다.
- 화면 공시는 전체 최근 공시를 보여줘 사용자가 직접 확인할 수 있게 한다.
- 편입 알림 공시는 좋은 재료 공시만 통과시킨다.
- 위험/부정 공시는 메인 선별 기준이 아니라 예외 경고 태그 후보로만 둔다.
- 공시는 전체 시장을 계속 훑는 구조가 아니라, 현재 후보 종목 기반으로 우선 추적한다.
- 차후에는 공시 선포착 후 거래대금/수급/가격 반응 확인 구조를 별도 축으로 추가할 수 있다.

## UI 작업 원칙

- UI를 예쁘게 정리하더라도 기존 데이터 바인딩과 이벤트 연결을 깨지 않는다.
- 탭으로 화면을 접고 재활용하는 작업에서는 매수, 매도, 잔고, 후보, 뉴스, 공시 영역의 연결 상태를 반드시 확인한다.
- 화면 표시용 값과 실제 판단용 값을 혼동하지 않는다.
- 화면 갱신 때문에 후보 행이 초기화되거나 순위가 흔들리면 안 된다.
- 사용자가 선택한 종목의 차트/정보만 실시간으로 갱신하는 구조를 존중한다.
